## 3) 정보구조(IA)와 페이지 섹션 순서

### IA 설계 원칙
- 흐름: **이해 → 신뢰 → 행동**
- 이탈 요인(정보 부족, 절차 불명확, 입력 피로) 단계별 제거
- CTA를 상단·중단·하단 반복 배치해 행동 경로 단축

### 페이지 섹션 순서
1. **헤더**
- 브랜드명, 핵심 메시지, 문의 CTA
2. **Hero**
- 가치 제안 헤드라인 + 보조 설명 + 1차 CTA
3. **핵심 강점 섹션**
- 차별 포인트 3~4개 카드
4. **문제-해결 섹션**
- 고객 불편 정의 → 해결 방식 제시
5. **진행 프로세스 섹션**
- 문의 접수 → 상담/확인 → 후속 안내
6. **신뢰 증빙 섹션**
- 후기, 운영 기준, 응답 정책
7. **FAQ 섹션**
- 비용/소요시간/진행 절차/문의 후 단계
8. **최종 CTA 섹션**
- 간편 문의 폼 + 예상 응답 시간
9. **푸터**
- 기본 정보, 연락 채널, 정책 링크

## 4) 시각 시스템(컬러/타이포/레이아웃/모션)

### 컬러 시스템
- **Primary**: Deep Navy (전문성·신뢰)
- **Secondary**: Teal 계열 (안정·친절)
- **Accent**: Amber/Orange (CTA 강조)
- **Neutral**: White/Light Gray (가독성·정돈감)

### 타이포그래피
- **헤드라인**: 굵은 산세리프, 짧은 핵심 문장
- **본문**: 16~18px, 높은 가독성 중심
- **보조 텍스트**: 13~14px, 안내/메타 정보 용도

### 레이아웃
- Desktop 12컬럼 / Mobile 1컬럼
- 상단: 메시지 집중
- 중단: 신뢰 및 설명 강화
- 하단: 전환 행동 집중
- 카드형 모듈로 정보 스캔 효율 강화

### 모션
- 절제된 마이크로 인터랙션
- 섹션 진입 fade-up
- 버튼 hover/focus 피드백
- 폼 오류/성공 상태 즉시 반영
- 과한 패럴랙스·자동 반복 애니메이션은 지양

## 5) 카피 전략(헤드라인 3안, CTA 5안)

### 카피 전략 방향
- “왜 지금 문의해야 하는가”를 명확하게 전달
- 신뢰 근거와 행동 유도를 한 흐름으로 연결
- CTA는 부담을 줄이는 안내형 문구로 설계

### 헤드라인 3안
1. **“문의가 쉬워지면, 전환은 자연스럽게 올라갑니다.”**
2. **“서울 고객이 신뢰하고 바로 선택하는 서비스.”**
3. **“복잡한 설명 없이, 필요한 답을 빠르게 안내합니다.”**

### CTA 5안
1. **무료 상담 문의하기**
2. **지금 문의 가능 여부 확인하기**
3. **우리 상황에 맞는 제안 받기**
4. **간편 문의 남기기**
5. **빠른 안내 요청하기**

## 6) 차별화 포인트(기존 흔한 디자인과 어떻게 다를지)

### 기존 흔한 랜딩의 문제
- 템플릿형 레이아웃 + 추상 카피 반복
- 시각 효과 과다 대비 신뢰 정보 부족
- CTA는 많지만 설득 근거가 약함
- 긴 입력 폼으로 이탈 증가

### 차별화 전략
1. **전환 중심 IA 설계**
- 미적 연출보다 문의 행동 흐름을 우선
2. **근거 중심 콘텐츠**
- 응답 기준/절차/지원 범위를 명확히 제시
3. **불안 해소 UX**
- FAQ + 프로세스로 문의 직전 망설임 제거
4. **모바일 우선 전환 구조**
- 반복 CTA + 최소 입력 폼
5. **절제된 신뢰형 비주얼**
- 정돈된 컬러·타이포·여백 중심 디자인
