## 3) 정보구조(IA)와 페이지 섹션 순서

### IA 원칙
- 흐름: **이해 → 신뢰 → 행동**
- 문의 이탈 요인(정보 부족, 절차 불명확, 입력 피로)을 단계별 해소
- CTA는 상단/중단/하단 반복 노출

### 페이지 섹션 순서
1. **헤더**: 브랜드명 + 핵심 메시지 + CTA
2. **Hero**: 가치 제안 헤드라인 + 보조 설명 + 1차 CTA
3. **핵심 강점**: 차별 포인트 3~4개
4. **문제-해결**: 고객 불편 정의 → 해결 방식
5. **진행 프로세스**: 문의 접수 → 상담/확인 → 후속 안내
6. **신뢰 증빙**: 후기/운영 기준/응답 정책
7. **FAQ**: 비용, 소요시간, 절차 관련 질문
8. **최종 CTA**: 간편 문의 폼 + 예상 응답 시간
9. **푸터**: 기본 정보, 정책, 연락 채널

## 4) 시각 시스템(컬러/타이포/레이아웃/모션)

### 컬러 시스템
- **Primary**: Deep Navy (전문성/신뢰)
- **Secondary**: Teal (안정/친절)
- **Accent**: Amber/Orange (CTA 강조)
- **Neutral**: White/Light Gray (가독성 중심)

### 타이포그래피
- **헤드라인**: 굵은 산세리프, 짧은 핵심 문장
- **본문**: 16~18px, 높은 가독성
- **보조 텍스트**: 13~14px, 안내/메타 정보

### 레이아웃
- Desktop 12컬럼 / Mobile 1컬럼
- 상단은 메시지 집중, 중단은 신뢰 강화, 하단은 전환 집중
- 카드형 모듈로 정보 스캔 효율 향상

### 모션
- 절제된 마이크로 인터랙션
- 섹션 진입 fade-up
- 버튼 hover/focus 시각 피드백
- 폼 오류/성공 상태 즉시 표시
- 과도한 패럴랙스·반복 애니메이션 배제

## 5) 카피 전략(헤드라인 3안, CTA 5안)

### 카피 전략 방향
- “왜 지금 문의해야 하는지”를 명확히 전달
- 신뢰 요소와 행동 유도를 동일 흐름에서 제시
- CTA는 강요가 아닌 안내형 문장으로 설계

### 헤드라인 3안
1. **“문의가 쉬워지면, 전환은 자연스럽게 올라갑니다.”**
2. **“서울 고객이 신뢰하고 바로 문의하는 서비스.”**
3. **“복잡한 설명 없이, 필요한 답을 빠르게.”**

### CTA 5안
1. **무료 상담 문의하기**
2. **지금 문의 가능 여부 확인하기**
3. **우리 상황에 맞는 제안 받기**
4. **간편 문의 남기기**
5. **빠른 안내 요청하기**

## 6) 차별화 포인트(기존 흔한 디자인과 어떻게 다를지)

### 흔한 랜딩의 문제
- 템플릿형 UI + 추상 카피 반복
- 시각 효과 과다 대비 신뢰 정보 부족
- CTA 반복은 많지만 설득 근거 부족
- 긴 폼으로 입력 피로 발생

### 차별화 전략
1. **전환 중심 IA 설계**
미적 연출보다 문의 행동 흐름 우선

2. **근거 중심 커뮤니케이션**
응답 기준, 절차, 지원 범위 명확히 제시

3. **불안 해소형 UX 구조**
FAQ + 프로세스로 문의 직전 망설임 제거

4. **모바일 우선 전환 최적화**
CTA 반복 노출 + 최소 입력 폼

5. **절제된 신뢰형 비주얼**
정돈된 컬러/타이포/여백으로 전문성 강화
