# DESIGN_SPEC.md

## 1) 브랜드 포지션/톤

### 브랜드 포지션
**오픈클로 신규 신청 테스트**는 서울 서비스업 맥락에서, 원페이지 안에서 신뢰를 형성하고 문의를 유도하는 브랜드로 포지셔닝한다.
핵심 목표는 “브랜드를 이해했다”에서 끝나지 않고 “지금 문의해도 되겠다”로 행동을 전환시키는 것이다.

- 핵심 가치
- **명확한 소개**: 첫 화면에서 서비스 정체성을 즉시 전달
- **신뢰 강화**: 특징/근거/상세 안내를 순차적으로 제공
- **전환 최적화**: CTA를 자연스럽게 반복 배치해 문의 유도

### 톤 & 매너
- 전문적이고 차분한 **안내형 톤**
- 과장보다 **근거 중심 카피**
- 압박형 문구 대신 **부담 낮은 제안형 CTA**
- 지양: 보장형 표현, 감성 과잉, 추상적 수식 반복

## 2) 핵심 사용자와 시나리오

### 핵심 사용자
1. **빠른 판단형 방문자**
- 첫 화면에서 신뢰 여부를 판단
2. **비교 검토형 방문자**
- 특징/프로세스/FAQ까지 읽고 문의 결정을 내림
3. **모바일 즉시행동형 방문자**
- 짧은 체류 내 문의를 완료하려는 사용자

### 대표 시나리오
- **시나리오 A (신규 유입)**
상단 메뉴 확인 → 대형 히어로로 브랜드 인지 → 특징/장점 확인 → 문의 버튼 클릭
- **시나리오 B (재검토 유입)**
특징 재확인 → 하단 프로세스/FAQ 확인 → 문의 폼 제출
- **시나리오 C (모바일 유입)**
히어로 CTA 또는 상단 ‘문의’ 메뉴 탭 → 폼 작성 → 완료

## 3) 정보구조(IA)와 페이지 섹션 순서
*(상단 메뉴 + 대형 히어로 + 특징/장점 + 상세설명 + footer)*

### 원페이지 구조
1. **Header (상단 고정)**
- 메뉴 2~3개: `소개 | 특징 | 문의`
- 우측 보조 CTA 버튼(문의하기)

2. **대형 Hero 섹션**
- 임팩트 있는 헤드라인 + 보조 카피
- 큰 사진/이미지 메인 비주얼
- 1차 CTA 버튼

3. **특징/장점 섹션 (중단 핵심)**
- 3~4개 핵심 장점 카드
- 각 카드: 장점 + 근거 + 기대효과

4. **신뢰 보강 블록 (권장)**
- 리뷰 요약, 성과 수치, 신뢰 배지(응답속도/만족도 등)

5. **상세설명 섹션 (하단)**
- **프로세스**: 문의 → 상담 → 진행
- **FAQ**: 비용/기간/응답시간/진행 방식
- **상세안내**: 제공 범위, 준비사항, 유의사항

6. **문의 섹션**
- 간편 문의 폼 + 예상 회신시간 안내
- 최종 CTA

7. **Footer (최하단)**
- 기본 정보, 연락 채널, 정책 링크

## 4) 시각 시스템(컬러/타이포/레이아웃/모션)

### 컬러
- **Primary**: Deep Navy (신뢰/전문성)
- **Secondary**: Teal (안정/친절)
- **Accent**: Amber (CTA/강조)
- **Neutral**: White/Light Gray (가독성)

### 타이포
- 헤드라인: 굵고 존재감 있는 산세리프
- 본문: 16~18px 중심의 읽기 쉬운 텍스트
- 보조텍스트: 13~14px 메타 정보용

### 레이아웃
- 원페이지 스크롤 기반
- 히어로는 **최소 80~100vh**로 크게 구성
- 특징 섹션은 카드형으로 시각적 구분 강화
- 상세설명은 스텝/아코디언형으로 정보 밀도 조절

### 모션
- 히어로 진입 시 텍스트-이미지 순차 등장
- 카드 hover/focus 시 미세 강조
- 섹션 전환은 짧은 fade-up
- 과한 애니메이션/패럴랙스는 지양

## 5) 카피 전략(헤드라인 3안, CTA 5안)

### 헤드라인 3안
1. **“신뢰를 먼저 보여주고, 문의를 자연스럽게 만듭니다.”**
2. **“한 페이지로 브랜드를 이해시키고, 행동까지 연결합니다.”**
3. **“복잡한 설명 없이, 믿고 문의할 수 있는 서비스.”**

### CTA 5안
1. **지금 문의하기**
2. **무료 상담 요청하기**
3. **빠른 안내 받기**
4. **간편 문의 남기기**
5. **우리 상황에 맞는 제안 받기**

## 6) 개발 handoff 체크리스트

### 구조/네비게이션
- [ ] 원페이지(one-page) 구조로 구현되었는가
- [ ] 상단 메뉴 2~3개(소개/특징/문의) 앵커 이동이 정상 동작하는가
- [ ] 최하단 footer 포함 여부 확인

### 섹션 구성
- [ ] 대형 히어로(큰 이미지 + 핵심 카피 + CTA) 구현
- [ ] 중단 특징/장점 섹션이 명확히 구분되는가
- [ ] 하단 상세설명(프로세스/FAQ/상세안내) 누락 없는가

### 전환 UX
- [ ] CTA가 히어로/중단/문의 섹션에 반복 배치되는가
- [ ] 문의 폼 필드 최소화 및 오류 메시지 처리 여부
- [ ] 제출 후 회신시간/다음 단계 안내가 노출되는가

### 반응형/접근성
- [ ] 모바일에서 메뉴/CTA 접근성 유지
- [ ] 텍스트 대비, 포커스 스타일, 터치 타겟 기준 충족
- [ ] FAQ 아코디언 키보드 접근 가능

### 성능/운영
- [ ] 히어로 이미지 최적화(WebP, lazy-load, 크기 지정)
- [ ] Core Web Vitals(LCP/CLS) 점검
- [ ] DNS 상태 확인 (`dns:created`)
- [ ] SEO/OG 메타 태그 반영
