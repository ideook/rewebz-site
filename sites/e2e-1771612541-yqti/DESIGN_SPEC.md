## 1) 브랜드 포지션/톤

### 브랜드 포지션
**“서울 고객이 빠르게 신뢰하고 바로 문의할 수 있는 서비스 브랜드”**

- 핵심 포지션: 화려한 인상보다 **명확한 정보 + 신뢰 + 빠른 응답 기대감**
- 랜딩 목표: 체류시간이 아니라 **문의 액션 전환율 개선**
- 브랜드 역할: 문제를 설명하는 업체가 아니라 **결정까지 돕는 파트너**

### 톤 앤 매너

- **문장 원칙**
- 짧고 이해 쉬운 문장
- 추상어 대신 구체적 안내(응답시간/진행절차/지원범위)
- 압박형 문구보다 안내형 문구
- **지양**
- 근거 없는 “최고/보장” 표현
- 과도한 긴급성/불안 조장 카피
- 기능 나열형 장문

## 2) 핵심 사용자와 시나리오

### 핵심 사용자
1. **빠른 결정형 사용자**
- 특징: 첫 인상에서 문의 여부 판단
- 니즈: 신뢰 근거 + 핵심 가치 즉시 파악
2. **비교 검토형 사용자**
- 특징: 여러 업체 비교 후 문의
- 니즈: 차별점, 프로세스 명확성, 상담 신뢰도
3. **모바일 즉시행동형 사용자**
- 특징: 짧은 체류, 빠른 입력 선호
- 니즈: 상시 CTA, 최소 입력 폼

### 사용자 시나리오
- **A. 신규 유입**
검색/광고 유입 → Hero 이해 → 강점/신뢰 확인 → 문의 제출
- **B. 재방문 검토**
FAQ/프로세스 확인 → 불안 해소 → CTA 클릭
- **C. 모바일 단기 체류**
상단 CTA 인지 → 간편 폼 입력 → 완료

## 3) 정보구조(IA)와 페이지 섹션 순서

### IA 원칙
- 전환 흐름: **이해 → 신뢰 → 행동**
- 문의 전환 장벽(정보 부족/절차 불명확/입력 피로) 단계별 해소
- CTA를 상단·중단·하단에 반복 배치

### 페이지 섹션 순서
1. **헤더** (브랜드명 + 핵심 한 줄 + CTA)
2. **Hero** (가치 제안 + 보조 설명 + 1차 CTA)
3. **핵심 강점 섹션** (차별 포인트 3~4개)
4. **문제-해결 섹션** (고객 페인포인트 → 해결 방식)
5. **진행 프로세스 섹션** (문의 → 상담 → 후속)
6. **신뢰 증빙 섹션** (후기/운영 원칙/응답 기준)
7. **FAQ 섹션** (비용, 소요시간, 진행 방식)
8. **최종 CTA 섹션** (간편 문의 폼 + 예상 응답 시간)
9. **푸터** (기본 정보/정책/연락채널)

## 4) 시각 시스템(컬러/타이포/레이아웃/모션)

### 컬러 시스템
- **Primary**: Deep Navy (신뢰·전문)
- **Secondary**: Teal 계열 (안정·친절)
- **Accent**: Amber 계열 (CTA 주목)
- **Neutral**: White/Light Gray (가독성 중심)

### 타이포 시스템
- **헤드라인**: 굵은 산세리프, 짧은 문장
- **본문**: 16~18px, 넉넉한 행간
- **보조**: 13~14px, 메타 정보 용도

### 레이아웃 시스템
- Desktop 12컬럼 / Mobile 1컬럼
- 상단은 메시지 집중, 중단은 설명·신뢰, 하단은 행동 유도
- 카드형 정보 모듈로 스캔 효율 강화

### 모션 원칙
- 절제된 마이크로 인터랙션
- 섹션 fade-up
- CTA hover/focus 강조
- 폼 오류/성공 즉시 피드백
- 과한 애니메이션 및 자동 반복 효과 지양

## 5) 카피 전략(헤드라인 3안, CTA 5안)

### 카피 전략 방향
- “왜 지금 문의해야 하는지”를 명확히 제시
- 신뢰 요소와 행동 유도를 한 문맥 안에서 연결
- CTA는 부담 낮은 안내형으로 설계

### 헤드라인 3안
1. **“문의가 쉬워지면, 전환은 자연스럽게 올라갑니다.”**
2. **“서울 고객이 신뢰하고 바로 문의하는 서비스 경험.”**
3. **“복잡한 설명 없이, 필요한 답을 빠르게.”**

### CTA 5안
1. **무료 상담 문의하기**
2. **지금 문의 가능 여부 확인하기**
3. **우리 상황에 맞는 제안 받기**
4. **간편 문의 남기기**
5. **빠른 안내 요청하기**

## 6) 차별화 포인트(기존 흔한 디자인과 어떻게 다를지)

### 흔한 랜딩의 문제
- 템플릿형 구성 + 추상 카피 반복
- 화려한 비주얼 대비 정보 신뢰 부족
- CTA는 많지만 문의 결정 근거 부족
- 폼이 길어 전환 이탈 발생

### 본 기획의 차별화
1. **전환 중심 IA 설계**: 예쁜 화면보다 행동 흐름 우선
2. **근거 중심 카피**: 응답/절차/지원 범위를 명확히 제시
3. **불안 해소 UX**: FAQ와 프로세스로 망설임 제거
4. **모바일 우선 전환 구조**: 반복 CTA + 최소 입력 폼
5. **절제된 신뢰형 비주얼**: 과장 없는 정돈감으로 전문성 강화
