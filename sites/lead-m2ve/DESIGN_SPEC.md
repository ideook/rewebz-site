## 1) 브랜드 포지션/톤

### 브랜드 포지션
**“신뢰를 먼저 설계해 리드를 전환시키는 서울 기반 서비스 브랜드”**

- 디자인의 목적을 ‘보여주기’가 아닌 **문의 전환**에 둔다.
- 사용자가 첫 스크린에서 “무엇을 하는 곳인지”와 “믿을 수 있는 이유”를 즉시 이해하도록 설계한다.
- 핵심 가치:
1. **명확한 제안**: 한 문장으로 핵심 가치 전달
2. **투명한 과정**: 문의 후 흐름이 예측 가능
3. **안정적 신뢰**: 과장 없는 근거 중심 커뮤니케이션

### 톤 앤 매너
- **톤**: 차분함 · 전문성 · 안내형 확신
- **문장 원칙**
- 짧고 단정한 문장
- 추상어보다 구체 정보(절차/응답시간/범위)
- 압박형 세일즈 대신 신뢰형 제안
- **지양**
- 보장·확정형 과장 카피
- 불안·긴급성 과다 유도
- 유행어 위주의 공허한 문장

## 2) 핵심 사용자와 시나리오

### 핵심 사용자
1. **빠른 판단형 실무자**
- 특성: 시간 부족, 첫 인상 중심 판단
- 니즈: 핵심 가치와 전문성의 즉시 확인

2. **비교 검토형 의사결정자**
- 특성: 복수 업체 비교 후 문의
- 니즈: 차별점, 응답 체계, 진행 방식의 명확성

3. **모바일 즉시행동형 사용자**
- 특성: 짧은 체류, 최소 입력 선호
- 니즈: 선명한 CTA, 빠른 폼 완료

### 사용자 시나리오
- **시나리오 A (신규 유입)**
검색/광고 유입 → Hero 가치 파악 → 신뢰 요소 확인 → 문의 제출
- **시나리오 B (재방문/비교 단계)**
FAQ·프로세스 재확인 → 불안 해소 → CTA 클릭
- **시나리오 C (모바일 단기 체류)**
상단 또는 고정 CTA 클릭 → 필수 항목 입력 → 문의 완료

## 3) 정보구조(IA)와 페이지 섹션 순서

### IA 설계 원칙
- 흐름: **인지 → 신뢰 → 행동**
- 문의 이탈 원인(정보 부족, 절차 불투명, 입력 피로)을 섹션별로 제거
- CTA를 주요 구간마다 반복 배치해 행동 경로 단축

### 페이지 섹션 순서
1. **헤더**
- 브랜드명 / 핵심 문장 / 1차 CTA
2. **Hero**
- 가치 제안 헤드라인 / 보조 설명 / 1차 문의 CTA
3. **핵심 강점 섹션**
- 차별 포인트 3~4개 카드형 요약
4. **문제-해결 섹션**
- 고객의 일반적 불편 → 해결 방법 제시
5. **프로세스 섹션**
- 문의 접수 → 진단/상담 → 실행/피드백
6. **신뢰 증빙 섹션**
- 후기, 운영 원칙, 응답 기준
7. **FAQ 섹션**
- 비용, 기간, 커뮤니케이션, 범위
8. **최종 CTA 섹션**
- 간편 문의 폼 + 예상 응답 시간
9. **푸터**
- 기본 정보, 연락 채널, 정책 링크

## 4) 시각 시스템(컬러/타이포/레이아웃/모션)

### 컬러 시스템
- **Primary**: Deep Navy (전문성/신뢰)
- **Secondary**: Teal 계열 (안정/친절)
- **Accent**: Amber 계열 (CTA 강조)
- **Neutral**: White/Light Gray (가독성/정돈감)

### 타이포그래피
- **헤드라인**: 굵은 산세리프, 짧은 핵심 문장
- **본문**: 16~18px, 읽기 편한 행간
- **보조 텍스트**: 13~14px, 설명·메타 정보

### 레이아웃
- Desktop 12컬럼 / Mobile 1컬럼
- 상단은 메시지 집중, 중단은 신뢰 정보 카드화, 하단은 전환 행동 집중
- 넉넉한 여백으로 프리미엄·안정감 확보

### 모션
- 절제된 마이크로 인터랙션
- 섹션 진입 시 fade-up
- 버튼 hover/focus 피드백
- 폼 오류/성공 상태 즉시 반영
- 과도한 패럴랙스/루프 애니메이션 지양

## 5) 카피 전략(헤드라인 3안, CTA 5안)

### 카피 전략 방향
- “왜 지금 문의해야 하는지”를 신뢰 기반으로 설득
- 근거(절차/응답/전문성)와 행동 유도를 한 흐름으로 연결
- CTA는 부담을 줄이는 안내형 문구 중심

### 헤드라인 3안
1. **“신뢰가 쌓이면, 리드 전환은 자연스럽게 올라갑니다.”**
2. **“서울 서비스업 맞춤, 문의가 이어지는 랜딩 설계.”**
3. **“보이는 디자인이 아니라, 믿고 문의하는 구조를 만듭니다.”**

### CTA 5안
1. **무료 진단 문의하기**
2. **지금 상담 가능 여부 확인하기**
3. **우리 상황에 맞는 제안 받기**
4. **간편 문의 남기기**
5. **빠른 안내 요청하기**

## 6) 차별화 포인트(기존 흔한 디자인과 어떻게 다를지)

### 기존 흔한 디자인의 한계
- 템플릿형 구성 + 추상 카피 반복
- 시각 효과는 많지만 신뢰 근거 부족
- CTA가 많아도 문의 결정을 돕는 정보가 약함
- 입력 항목 과다로 폼 이탈 발생

### 차별화 전략
1. **전환 중심 IA**
- 미적 연출보다 문의 행동 흐름을 우선
2. **근거 중심 카피**
- 응답 기준·절차·지원 범위를 명확히 제시
3. **불안 해소 UX**
- FAQ + 프로세스로 문의 직전 망설임 제거
4. **모바일 우선 설계**
- 반복 CTA + 최소 입력 폼
5. **절제된 프리미엄 톤**
- 정돈된 컬러·타이포·여백으로 신뢰 강화
