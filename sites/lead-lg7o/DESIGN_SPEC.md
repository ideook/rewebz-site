## 1) 브랜드 포지션/톤

### 브랜드 포지션
**“서울 고객이 빠르게 이해하고 바로 문의할 수 있는 신뢰형 서비스 브랜드”**

- 원페이지에서 브랜드 소개와 문의 전환을 한 번에 완결하는 구조
- 핵심 가치:
1. **즉시 이해**: 첫 화면에서 무엇을 하는지 명확
2. **신뢰 형성**: 특징/근거/상세 안내로 망설임 해소
3. **행동 유도**: 문의 CTA를 자연스럽게 연결

### 톤 앤 매너
- **톤**: 전문적, 간결, 안내형
- **문장 원칙**
- 짧고 명확한 문장
- 추상어보다 구체 정보 중심
- 압박형 세일즈 대신 제안형 카피
- **지양**
- 과장/보장형 문구
- 불안 조장 표현
- 장문 설명 위주의 구성

## 2) 핵심 사용자와 시나리오

### 핵심 사용자
1. **첫 방문 빠른 판단형**
- 니즈: 브랜드 신뢰성과 제공 서비스 즉시 파악
2. **비교 검토형**
- 니즈: 장점/프로세스/FAQ 확인 후 문의 결정
3. **모바일 즉시 문의형**
- 니즈: 짧은 스크롤 동선에서 바로 문의 가능

### 사용자 시나리오
- **A. 신규 유입**
상단 메뉴 인지 → 대형 히어로 메시지 확인 → 특징/장점 확인 → 문의 CTA 클릭
- **B. 검토 유입**
특징 확인 → 하단 상세 설명(프로세스/FAQ) 확인 → 문의 폼 제출
- **C. 모바일 유입**
히어로 CTA 또는 메뉴 ‘문의’ 탭 → 폼 작성 → 완료

## 3) 정보구조(IA)와 페이지 섹션 순서
*(원페이지 구성 기준)*

1. **상단 헤더/메뉴 (2~3개)**
- 메뉴 예시: `소개 | 특징 | 문의`
- 우측 보조 CTA 버튼(문의하기)

2. **대형 히어로 섹션**
- 강한 헤드라인 + 서브카피 + 1차 CTA
- 메인 비주얼: 큰 사진/이미지 중심(브랜드 신뢰감 전달)

3. **특징/장점 섹션 (중단 핵심)**
- 3~4개 핵심 장점 카드
- 각 장점에 근거성 보조 문장 추가

4. **신뢰 보강 섹션 (권장)**
- 리뷰/성과 숫자/배지 하이라이트 블록
- 문의 전 마지막 확신 제공

5. **하단 상세 설명 섹션**
- **프로세스**: 문의 → 상담 → 진행
- **FAQ**: 비용, 기간, 응답, 진행 방식
- **상세안내**: 서비스 범위, 유의사항, 준비 항목

6. **최종 문의 섹션**
- 간편 문의 폼 + 예상 응답시간 안내

7. **Footer**
- 회사 기본 정보, 연락 채널, 정책 링크

## 4) 시각 시스템(컬러/타이포/레이아웃/모션)

### 컬러 시스템
- **Primary**: Deep Navy (신뢰/전문)
- **Secondary**: Teal 계열 (안정/친절)
- **Accent**: Amber 계열 (CTA 강조)
- **Neutral**: White/Light Gray (가독성 중심)

### 타이포그래피
- **헤드라인**: 굵은 산세리프, 짧은 문장
- **본문**: 16~18px, 읽기 편한 행간
- **보조 텍스트**: 13~14px, 설명/메타 정보

### 레이아웃
- 원페이지 스크롤 구조
- 히어로는 크게(최소 80~100vh) 구성
- 특징 섹션은 카드형으로 한눈에 보이게 배치
- 하단 상세 설명은 스텝/아코디언 구조로 정리

### 모션
- 히어로 진입 시 텍스트+이미지 순차 등장
- 카드 hover/focus 시 미세 강조
- 섹션 전환은 짧은 fade-up
- 과도한 애니메이션은 지양
