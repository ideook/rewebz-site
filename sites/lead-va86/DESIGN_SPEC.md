## 1) 브랜드 포지션/톤

### 브랜드 포지션
**“서울 기반 서비스 브랜드로서, 한 페이지 안에서 신뢰를 전달하고 문의까지 연결하는 원페이지 브랜드”**

- 페이지 목적: 정보 분산 없이 **브랜드 소개 + 문의 전환**을 한 흐름으로 완결
- 핵심 가치:
1. **명확한 소개**: 처음 5초 안에 브랜드 이해
2. **신뢰 강화**: 특징/장점과 상세 안내로 불안 해소
3. **행동 유도**: CTA를 자연스럽게 반복 노출

### 톤 앤 매너
- **톤**: 전문적, 친절한 안내형, 과장 없는 자신감
- **문장 원칙**
- 짧고 직관적인 문장
- 추상어보다 구체 정보(절차/응답/범위)
- 압박형보다 제안형 CTA
- **지양**
- 과장형 보장 문구
- 감성 과잉 표현
- 기능 나열형 장문

## 2) 핵심 사용자와 시나리오

### 핵심 사용자
1. **첫 방문 빠른 판단형**
- 니즈: “이 브랜드가 어떤 곳인지” 즉시 파악
2. **비교 검토형**
- 니즈: 특징/장점과 상세 프로세스로 신뢰 확인
3. **모바일 즉시 문의형**
- 니즈: 짧은 스크롤 안에서 CTA 접근, 간편 문의

### 사용자 시나리오
- **A. 신규 유입**
상단 메뉴 인지 → 대형 히어로 메시지 확인 → 특징 섹션 탐색 → 문의 CTA 클릭
- **B. 검토 유입**
특징/장점 확인 → 하단 상세 설명(프로세스/FAQ) 확인 → 문의 제출
- **C. 모바일 유입**
Hero CTA 또는 메뉴 ‘문의’ 이동 → 폼 작성 → 완료

## 3) 정보구조(IA)와 페이지 섹션 순서
*(원페이지 one-page 구성)*

### 상단 메뉴(2~3개)
- **소개**
- **특징**
- **문의**
> 상단 메뉴 클릭 시 해당 섹션 앵커로 이동

### 섹션 순서 (Top → Bottom)
1. **Header / Top Navigation**
- 로고 + 메뉴 2~3개 + 우측 CTA 버튼
2. **대형 Hero 섹션**
- 강한 헤드라인
- 보조 카피
- 메인 비주얼(대형 사진/이미지)
- 1차 CTA
3. **특징/장점 섹션**
- 3~4개 핵심 장점 카드
- 각 장점에 근거/효과 한 줄 추가
4. **신뢰 보강 섹션 (선택 권장)**
- 리뷰/성과 수치/배지 하이라이트
5. **하단 상세 설명 섹션**
- **프로세스** (문의 → 상담 → 진행)
- **FAQ** (비용/기간/응답/진행 방식)
- **상세 안내** (서비스 범위/유의사항)
6. **문의 CTA 섹션**
- 간편 폼 + 예상 응답시간
7. **Footer**
- 기본 정보, 연락 채널, 정책 링크

## 4) 시각 시스템(컬러/타이포/레이아웃/모션)

### 컬러 시스템
- **Primary**: Deep Navy (신뢰/전문성)
- **Secondary**: Teal 계열 (안정/친절)
- **Accent**: Amber 계열 (CTA 강조)
- **Neutral**: White/Light Gray (가독성)

### 타이포그래피
- **헤드라인**: 굵은 산세리프, 임팩트 있는 짧은 문장
- **본문**: 16~18px, 가독성 중심
- **보조 정보**: 13~14px

### 레이아웃
- 원페이지 스크롤 구조
- Hero는 화면 상단에서 가장 큰 볼륨(최소 80~100vh 권장)
- 특징 섹션은 카드형 2~4열(모바일 1열)
- 상세설명은 아코디언/스텝 구조로 정보 정리

### 모션
- Hero 진입 시 텍스트와 비주얼 순차 등장
- 카드 hover/focus 미세 강조
- 섹션 전환은 짧은 fade-up
- 과한 패럴랙스/지속 애니메이션 지양

## 6) 개발 handoff 체크리스트

### 구조/콘텐츠
- [ ] 원페이지(one-page) 구조로 구현되었는가
- [ ] 상단 메뉴 2~3개(소개/특징/문의) 앵커 이동이 동작하는가
- [ ] 대형 히어로 + 특징/장점 + 상세설명 + footer가 모두 포함되었는가

### UI/컴포넌트
- [ ] Hero 메인 비주얼이 충분히 크게 노출되는가
- [ ] 특징/장점 카드 컴포넌트가 일관되게 적용되는가
- [ ] CTA 버튼 상태(기본/hover/focus) 정의 완료

### 전환 UX
- [ ] 히어로/중단/하단에 CTA가 반복 배치되었는가
- [ ] 문의 폼 필드 최소화 및 오류 처리 메시지가 있는가
- [ ] 제출 후 응답 시간/다음 단계 안내가 제공되는가

### 반응형/접근성
- [ ] 모바일에서 메뉴/CTA 접근성이 유지되는가
- [ ] 텍스트 대비, 포커스 가시성, 터치 타겟 기준을 충족하는가
- [ ] FAQ/아코디언 사용 시 키보드 접근이 가능한가

### 성능/운영
- [ ] 메인 비주얼 이미지 최적화(WebP/지연 로딩) 적용
- [ ] Core Web Vitals(LCP/CLS) 점검
- [ ] DNS 상태 확인 (`dns:created`)
- [ ] SEO/OG 메타 반영 여부 확인

## 최종 요약
원페이지 프롬프트 재테스트는 **상단 간결 메뉴 + 대형 히어로 + 특징/장점 + 하단 상세설명 + 문의 CTA + footer**를 축으로,
한 페이지 내에서 브랜드 이해와 신뢰 형성을 완료하고 문의 전환까지 연결하는 원페이지 전략으로 설계한다.
