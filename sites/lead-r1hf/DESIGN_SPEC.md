# DESIGN_SPEC.md

## 1) 브랜드 포지션/톤

### 브랜드 포지션
**오픈클로 테스트 스튜디오**는 서울 기반 서비스 브랜드로서,
한 페이지 안에서 **브랜드 신뢰 형성 → 문의 행동 전환**까지 완결하는 구조를 지향한다.

- 핵심 가치
1. **명확한 소개**: 첫 화면에서 “무엇을 하는 브랜드인지” 즉시 이해
2. **신뢰 강화**: 특징·근거·상세 안내로 불안 제거
3. **문의 유도**: 자연스러운 CTA 흐름으로 전환 촉진

### 톤 앤 매너
- **톤**: 전문적 · 차분함 · 안내형
- **문장 원칙**
- 짧고 단정한 문장
- 추상어보다 구체 정보(절차/응답시간/범위)
- 강요보다 제안 중심
- **지양**
- 과장/보장형 표현
- 과도한 감성 카피
- 기능 나열형 장문

## 2) 핵심 사용자와 시나리오

### 핵심 사용자
1. **첫 방문 빠른 판단형**
- 니즈: 신뢰할 수 있는 브랜드인지 빠르게 판단
2. **비교 검토형**
- 니즈: 특징/장점, 프로세스, FAQ 확인 후 문의 결정
3. **모바일 즉시문의형**
- 니즈: 짧은 동선에서 바로 문의 가능해야 함

### 사용자 시나리오
- **A. 신규 유입**
- 상단 메뉴 인지 → 대형 히어로 확인 → 특징/장점 탐색 → 문의 CTA 클릭
- **B. 검토 유입**
- 장점 확인 → 하단 상세설명(프로세스/FAQ/상세안내) 확인 → 문의 제출
- **C. 모바일 유입**
- 메뉴 ‘문의’ 또는 히어로 CTA로 이동 → 간편 폼 작성 → 완료

## 3) 정보구조(IA)와 페이지 섹션 순서 (상단 메뉴 + 대형 히어로 + 특징/장점 + 상세설명 + footer)

> **원페이지(one-page) 고정 구조**

1. **Header / Top Menu**
- 메뉴 2~3개: `소개 | 특징 | 문의`
- 우측 보조 CTA(문의하기)

2. **대형 Hero 섹션**
- 강한 헤드라인 + 서브카피
- 메인 비주얼(대형 사진/이미지) 중심
- 1차 CTA 버튼

3. **특징/장점 섹션 (중단 핵심)**
- 3~4개 카드형 장점
- 각 카드: 장점 + 근거 + 기대 효과

4. **신뢰 보강 블록 (중간 하이라이트)**
- 배지(응답 속도/진행 건수/만족도)
- 통계/리뷰/성과 요약 블록

5. **상세설명 섹션 (하단)**
- **프로세스**: 문의 → 상담 → 실행/안내
- **FAQ**: 비용, 기간, 응답, 범위
- **상세안내**: 준비사항, 진행 방식, 유의점

6. **문의 섹션**
- 간편 폼 + 예상 응답시간 안내
- 최종 CTA

7. **Footer (최하단)**
- 기본 정보, 연락 채널, 정책 링크

## 4) 시각 시스템(컬러/타이포/레이아웃/모션)

### 컬러 시스템
- **Primary**: Deep Navy (신뢰/전문)
- **Secondary**: Teal (안정/친절)
- **Accent**: Amber (CTA 강조)
- **Neutral**: White / Light Gray (가독성)

### 타이포그래피
- **헤드라인**: 볼드 산세리프, 임팩트 중심
- **본문**: 16~18px, 높은 가독성
- **보조 텍스트**: 13~14px

### 레이아웃
- 원페이지 스크롤 구조
- Hero는 최소 80~100vh 권장
- 특징/장점은 카드형으로 한눈에 구분
- 상세설명은 스텝/아코디언형으로 정보 밀도 조절

### 모션
- 히어로 진입: 카피/비주얼 순차 등장
- 카드 hover/focus 미세 강조
- 섹션 전환 fade-up
- 과한 패럴랙스/지속 애니메이션은 지양

## 5) 카피 전략(헤드라인 3안, CTA 5안)

### 헤드라인 3안
1. **“한 페이지로 신뢰를 만들고, 문의까지 연결합니다.”**
2. **“브랜드를 명확히 보여주고, 행동을 이끌어내는 원페이지.”**
3. **“처음 보는 고객도 이해하고 문의하게 만드는 구조.”**

### CTA 5안
1. **지금 문의하기**
2. **무료 상담 요청하기**
3. **빠른 안내 받기**
4. **간편 문의 남기기**
5. **맞춤 제안 받아보기**

## 6) 개발 handoff 체크리스트

### 구조/네비게이션
- [ ] 원페이지(one-page)로 구현되었는가
- [ ] 상단 메뉴 2~3개(소개/특징/문의) 앵커 이동이 동작하는가
- [ ] 최하단 footer 포함 여부 확인

### 콘텐츠/섹션
- [ ] 대형 히어로(큰 이미지 + 핵심 카피) 구현 완료
- [ ] 중단 특징/장점 섹션이 시각적으로 명확한가
- [ ] 하단 상세설명(프로세스/FAQ/상세안내) 누락 없는가

### 전환 UX
- [ ] 히어로/중단/문의 섹션에 CTA가 반복 배치되었는가
- [ ] 문의 폼 필드 최소화 및 오류 메시지 처리되었는가
- [ ] 제출 후 응답 시간/다음 단계 안내가 표시되는가

### 반응형/접근성
- [ ] 모바일에서 메뉴/CTA 접근성이 유지되는가
- [ ] 텍스트 대비, 포커스, 터치 타겟 기준 충족
- [ ] FAQ 아코디언 키보드 접근 가능 여부 확인

### 성능/운영
- [ ] 히어로 이미지 최적화(WebP, lazy-load, 사이즈 지정)
- [ ] Core Web Vitals(LCP/CLS) 점검
- [ ] DNS 상태 확인 (`dns:created`)
