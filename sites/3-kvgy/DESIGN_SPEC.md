## 2) 핵심 사용자와 시나리오

### 핵심 사용자
1. **첫 방문 빠른 판단형**
- “이 브랜드가 믿을 만한가?”를 5~10초 내 판단
2. **비교 검토형**
- 특징/장점, 프로세스, FAQ 확인 후 문의 결정
3. **모바일 즉시 문의형**
- 짧은 체류 시간 내 CTA 클릭과 폼 제출 선호

### 사용자 시나리오
- **A. 신규 유입**
- 상단 메뉴 확인 → 대형 히어로에서 가치 인지 → 특징/장점 확인 → 문의 CTA 클릭
- **B. 검토 유입**
- 중단 특징 확인 → 하단 상세설명(프로세스/FAQ/상세안내) 검토 → 문의 제출
- **C. 모바일 유입**
- 메뉴 ‘문의’ 탭 또는 히어로 CTA 클릭 → 간편 폼 제출 → 완료 안내 확인

## 3) 정보구조(IA)와 페이지 섹션 순서
*(상단 메뉴 + 대형 히어로 + 특징/장점 + 상세설명 + footer)*

> **원페이지(one-page) 구성**

1. **Header (상단 고정)**
- 메뉴 2~3개: `소개 | 특징 | 문의`
- 우측 CTA 버튼(문의하기)

2. **대형 Hero 섹션**
- 강한 헤드라인 + 신뢰 보조 카피
- 메인 비주얼(큰 사진/이미지) 중심
- 1차 CTA

3. **특징/장점 섹션 (중단 핵심)**
- 3~4개 장점 카드
- 구성: 장점명 + 근거 + 기대효과

4. **신뢰 보강 블록 (권장)**
- 배지/통계/리뷰 하이라이트
- 예: 평균 응답시간, 진행 건수, 고객 만족도

5. **하단 상세설명 섹션**
- **프로세스**: 문의 → 상담 → 진행/결과 안내
- **FAQ**: 비용, 기간, 응답 방식, 진행 범위
- **상세안내**: 준비사항, 유의사항, 커뮤니케이션 방식

6. **문의 섹션**
- 간편 문의 폼 + 예상 회신시간 + 최종 CTA

7. **Footer (최하단)**
- 회사 기본 정보, 연락 채널, 정책 링크

## 4) 시각 시스템(컬러/타이포/레이아웃/모션)

### 컬러 시스템
- **Primary**: Deep Navy (신뢰/전문성)
- **Secondary**: Teal (안정/친절)
- **Accent**: Amber (CTA/강조)
- **Neutral**: White/Light Gray (가독성)

### 타이포그래피
- **헤드라인**: 볼드 산세리프 (강한 인지)
- **본문**: 16~18px 중심 (읽기 편한 행간)
- **보조 텍스트**: 13~14px (메타/설명)

### 레이아웃
- 원페이지 수직 스크롤
- 히어로는 최소 80~100vh로 크게 구성
- 특징/장점은 카드형 그리드로 빠른 스캔 가능
- 상세설명은 스텝/아코디언형으로 정보 정리

### 모션
- 히어로 진입 시 텍스트→비주얼→CTA 순차 등장
- 카드 hover/focus 미세 강조
- 섹션 전환은 짧은 fade-up
- 과한 패럴랙스/반복 애니메이션은 배제
